# 說話等級大改版：轉生系統設計

## 背景與痛點

現況：
- 等級上限 173（由 `chat_exp_unit` table 驅動，非程式碼硬編），但 title 系統實際只覆蓋到 Level 80，81–173 顯示會 crash
- 已有 1 位用戶達到上限（XP 甚至超出 table 範圍），82 位用戶在 Level 100 以上
- 平均等級 16.9，絕大多數用戶仍在低等
- 娛樂性不足：到頂後無事可做
- 稱號被等級綁死，無個人化空間

## 核心設計：轉生 + 試煉系統

### 等級上限調整

- 新轉生門檻：**Level 100**
- 轉生後等級歸零，重新爬回 100
- 上限數字本身不硬編，由 `prestige_trials` table 的資料筆數決定（目前規劃 5 次）

### 轉生流程

```
轉生（需 Lv.100）
  → 等級歸零
  → 試煉限制立即生效
  → 爬回 Lv.100
  → 限制解除 + 永久獎勵立即解放
  → 自由決定是否繼續下一次轉生
```

試煉完成時機：**重新滿等（Lv.100）當下立即達成**，不需要再次轉生才兌現。

### 試煉設計（目前規劃，資料驅動非硬編）

| 轉生次數 | 試煉內容 | 完成後永久解放 |
|---|---|---|
| ★1 | 無限制，反而 XP +20%（蜜月期） | 稱號自選功能解鎖 |
| ★2 | XP 獲取倍率 -30% | 永久 XP 倍率 +10% |
| ★3 | 訊息冷卻嚴格化（需 8 秒才算滿速） | 冷卻時間永久縮短 |
| ★4 | 群組人數加成停用 | 群組加成翻倍 |
| ★5 | XP 獲取倍率 -50%（最終試煉） | 永久 XP 倍率 +15% + 限定稱號 |

完成全部 ★5 後進入「覺醒」狀態，累積 XP 倍率從基礎 90 提升至約 122（+35%）。

### 群組可見性

- **冒險小卡**：試煉中顯示 `⚔️ ★N 試煉進行中` 及限制說明；覺醒後顯示 `✨ 覺醒者`
- **群組廣播**：
  - 轉生開始：`「[用戶名] 踏入了 ★N 的試煉」`
  - 試煉完成：`「[用戶名] 通過了 ★N 的試煉，永久解放 XXX」`

### 稱號系統

- ★1 完成後解鎖稱號自選（已收集的稱號可自由選擇顯示）
- ★5 完成後解鎖限定稱號（覺醒者專屬）
- 現有的 `chat_level_title` + `chat_range_title` 需要補完至 Level 100

## 需要變更的範圍（待規劃）

- DB：新增 `prestige_trials` table（試煉設定）、`user_prestige` table（用戶轉生狀態）
- `chat_user_data`：可能需要新增 prestige 相關欄位
- `chat_exp_unit`：調整 XP 曲線至 Level 100
- `chat_level_title`：補完 Level 1–100 的稱號覆蓋
- `ChatLevelController`：新增轉生指令、狀態卡顯示更新
- `EventDequeue.js`：XP 計算加入試煉限制
- 群組廣播邏輯
- 前端 Rankings 頁面：顯示轉生次數
