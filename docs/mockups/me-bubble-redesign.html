v3 · Dark Luxury 訂閱卡

  • 訂閱顏色全面更換：舊的 indigo #3D52D5 / rose #B88C9E 不延續， 改為深 navy #12243A（theme.js HERO_SURFACE.bgAlt），跟 Janken 段位卡 / 戰王類 bubble 一致。
  • 金色細線：頂部 3px amber 漸層 hairline（#F59E0B → #FCD34D → #F59E0B）， 視覺上像一條金框，傳達「高級感 / 訂閱會員」訊號。
  • 月卡 vs 季卡用 tag 區分：月卡 tag 用 cyan，季卡 tag 用 amber；tag 反白為深色字 → 「polished badge」的感覺。 內文底色不變，保持 dark navy 的沉穩。
  • Bullet 點改菱形 ◆（琥珀色），不再用普通 bullet •，視覺更精緻。
  • Stats card 左側色條（3px）：cyan / green / red 分別對應進行中 / 已完成 / 未完成， 跟 Progress bubble 的 accent bar 呼應，把 Profile 各 section 串成一致的 card 系統。
  • size 退回 mega：giga 是治標。Dark luxury 設計把視覺重量拉低，Profile 自然就不擠了。

v2 修正摘要

  • 對比度（必修）：Hero 背景改 linear-gradient(#00838F → #00ACC1)。 白字 on #00838F 對比 ≈ 5.3:1，WCAG AA 通過。Level / rank 都安全可讀。
  • LINE Flex 相容性：avatar 外框改 2px 實色白，去除 box-shadow（LINE 不支援）。 stat card / wallet / janken row 全部從 CSS grid 改成 flex，對應 layout: "horizontal" + flex:1。
  • 身份資訊 2 行化： name + Lv.12 · 初級 pill 一行，Rank #42 一行。 原本的「初級 的 見習生」字串被壓縮進等級 pill，rank 用琥珀色強調。 （稱號/成就功能正重構中，先不放。）
  • Stat card 格式統一：全部 icon + 分數（✓ 1/1 ｜ ! 0/1 ｜ ⋯ 3/7）， icon 提供非顏色線索，解決色盲依賴問題。
  • Progress 少一 block：「上次出彩/出新」收進對應進度條的 block-meta， bubble 高度降低。新增頂部 4px cyan accent bar，跟 Profile 呼應。
  • 文案統一中文：Progress pill 的 PROGRESS 改成「蒐集 · 戰績」副標。 底部 LIFF CTA 暫時移除（等未來 /me 頁做完再補）。

訂閱顯示策略（閾值拆分）

  • 0 張卡：Profile 不顯示 sub-panel。
  • 1 張卡：sub-panel 內嵌在 Profile（hero 與 stats 之間），顯示到期日與效果清單。
  • 2 張卡（月+季）：Profile 內嵌會衝到 ~410px，超過 LINE 舒適範圍。 建議改成 3-bubble carousel（Profile → 訂閱特權 → Progress）， Profile 改放 sub-badge 小色塊當提示（「🎟 訂閱中 · 月卡 + 季卡」）， 詳細效果搬到獨立 Subscription bubble。

實作待辦

  • 移除 ChatLevelController.showStatus 末段的 context.replyText("尚未有任何數據，經驗開始累積後即可投胎！")。 新 UI 的空狀態（Level 0 + 空 EXP bar + 0/200 蒐集 + 0 勝 0 敗）已經能表達「尚未投胎」。

空狀態處理原則

  • 猜拳無紀錄：顯示 0 勝 0 敗 0 平 · 勝率 -%，不收 block。
  • 轉蛋無紀錄：120/200 顯示 0/200、上次出彩 顯示 -。
  • 連續簽到 0 天：顯示 0 天，不收 row。
  • 訂閱 0 張：整個 sub-panel / sub-badge 區塊不渲染（hero 直接接 stats）。
  • 適用原則：所有統計數值的預設值是「0」或「-」，不要整段隱藏， 讓使用者看到「我在這個系統還沒有數據」而不是「這個系統不存在」。 其他 feature bubble（競技場、戰王、市集…）也比照辦理。
